# Bloom Marketing Website - UX Strategy

## Executive Summary
Mobile-first marketing website optimized for converting PC gamers into Steam wishlists. Targets 60%+ mobile traffic with accessible, fast-loading design that balances aspiration with transparency about Early Access scope.

---

## 1. User Research & Personas

### Primary Persona: "The Tactical Co-op Gamer"
- **Age**: 25-35
- **Games Played**: Deep Rock Galactic, Escape from Tarkov, Valheim
- **Motivations**: Cooperative strategy, progression systems, replayability
- **Device**: 65% mobile (discovery), 35% desktop (deep research)
- **Pain Points**: Skeptical of Early Access promises, wants concrete feature lists
- **Conversion Trigger**: Clear co-op mechanics + honest roadmap

### Secondary Persona: "The Extraction Shooter Enthusiast"
- **Age**: 20-28
- **Games Played**: Hunt: Showdown, Marauders, Cycle: Frontier
- **Motivations**: High-stakes gameplay, competitive edge, mastery
- **Device**: 55% mobile, 45% desktop
- **Pain Points**: Genre fatigue, wants unique differentiator
- **Conversion Trigger**: Faction system novelty + PvE focus

## 3. Design System Foundation

### Color Palette
```
Primary Background: #0a0f14 (Deep Space Black)
Secondary Background: #151d26 (Charcoal)
Surface: #1e2936 (Slate)

Accent Primary: #00d9ff (Cyan) - CTAs, links, highlights

Text Secondary: #b8c5d6 (Steel Blue) - Body text

### Typography Scale
```

## 9. Accessibility Checklist

### 9.1 Keyboard Navigation
- [ ] All interactive elements focusable via Tab
- [ ] Logical tab order (left-to-right, top-to-bottom)
- [ ] Skip to main content link (first focusable element)
- [ ] Modal dialogs trap focus
- [ ] ESC key closes modals/menus
- [ ] Enter/Space activates buttons
- [ ] Arrow keys navigate carousels/tabs

### 9.2 Screen Readers
- [ ] Semantic HTML (header, nav, main, article, footer)
- [ ] ARIA landmarks (role="navigation", "main", etc.)
- [ ] ARIA labels for icon-only buttons
- [ ] Alt text for all images (descriptive, not "image of")
- [ ] Empty alt for decorative images (alt="")
- [ ] Form labels associated with inputs (for/id)
- [ ] Error messages announced (aria-live="polite")
- [ ] Loading states announced (aria-busy="true")

### 9.3 Visual Accessibility
- [ ] Color contrast 4.5:1 minimum (text)
- [ ] Color contrast 3:1 minimum (UI components)
- [ ] Don't rely on color alone (use icons + text)
- [ ] Text resizable to 200% without breaking layout
- [ ] Focus indicators visible (2px cyan outline)
- [ ] Hover states don't hide content

### 9.4 Motion & Animation
- [ ] Respect prefers-reduced-motion
  - Disable parallax
  - Disable autoplay
  - Reduce animation duration
  - Remove sliding/scaling transitions
- [ ] No autoplay video with sound
- [ ] Pause/stop controls for carousels
- [ ] No flashing content (seizure risk)

### 9.5 Mobile Accessibility
- [ ] Touch targets minimum 44x44px
- [ ] Pinch-to-zoom enabled (no viewport lock)
- [ ] Landscape orientation supported
- [ ] Text readable without zoom (16px minimum)
- [ ] Forms use appropriate input types (email, tel, etc.)

## 10. Performance Optimization

### 10.1 Core Web Vitals Targets
- **LCP (Largest Contentful Paint)**: < 2.5s
  - Hero image: WebP format, < 200KB
  - Above-fold content prioritized
  - Preload critical resources

- **FID (First Input Delay)**: < 100ms
  - Minimal JavaScript on initial load
  - Defer non-critical scripts

- **CLS (Cumulative Layout Shift)**: < 0.1
  - Reserve space for images (aspect-ratio)
  - Fixed header height
  - No layout shift on font load

### 10.2 Image Optimization
- **Format**: WebP with JPEG fallback
- **Lazy loading**: Below-fold images
- **Responsive images**: srcset for different sizes
- **Compression**: 80% quality, < 150KB per image
- **CDN**: Cloudflare/Vercel for global delivery

### 10.3 Code Optimization
- **CSS**:
  - Critical CSS inlined
  - Non-critical CSS deferred
  - Minified + gzipped

- **JavaScript**:
  - Code splitting by route
  - Tree shaking unused code
  - Async/defer non-critical scripts

- **Fonts**:
  - Subset fonts (Latin characters only)
  - font-display: swap
  - Preload font files

### 10.4 Caching Strategy
- **Static assets**: Cache 1 year
- **HTML**: Cache 5 minutes
- **API responses**: Cache 1 hour (if applicable)
- **Service worker**: Offline fallback page

## 16. Risk Mitigation

### Technical Risks
**Risk**: Video background causes performance issues on mobile
**Mitigation**:
- Serve static image to mobile devices
- Lazy load video on desktop
- Provide toggle to disable animations

**Risk**: Image-heavy design slows page load
**Mitigation**:
- Aggressive lazy loading
- WebP format with JPEG fallback
- CDN delivery
- Responsive images (srcset)

### UX Risks
**Risk**: Users don't understand faction system
**Mitigation**:
- Clear "What are Factions?" explainer on Factions page
- Video demo of faction abilities
- Infographic comparing to traditional classes

**Risk**: Early Access skepticism reduces wishlists
**Mitigation**:
- Transparent EA scope messaging on homepage
- Dedicated FAQ section for EA concerns
- Roadmap visualization showing post-launch content
- Honest tone in all copy

### Accessibility Risks
**Risk**: Complex interactions not keyboard-accessible
**Mitigation**:
- Test all interactions with keyboard-only navigation
- ARIA labels for dynamic content
- Focus management for modals/menus

**Risk**: Dark theme creates contrast issues
**Mitigation**:
- Test all text against backgrounds (4.5:1 minimum)
- Provide high-contrast mode toggle (future)
- Use cyan accent strategically for readability
